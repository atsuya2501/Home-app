# ミニマルランチャー

スマホ依存対策のための、許可したアプリだけをホーム画面に表示するシンプルなランチャー。個人利用専用。

## 機能
- 選んだアプリだけをホーム画面へ表示
- アプリのフォルダ分け、ホーム・フォルダ内・下段ドックのドラッグ並べ替え
- SNSなど任意のアプリに、起動理由入力または5秒待機のゲートを設定
- 起動理由の履歴表示（30日を過ぎた履歴は自動削除）
- アプリ名検索付きの設定画面
- 壁紙に合わせたホーム文字・ステータス表示色の自動切り替え（白・黒の手動指定も可能）
- 配置・設定・起動理由履歴のJSONバックアップ／復元
- 設定は端末内（SharedPreferences）に保存（Room・Hilt不使用）
- Androidのデフォルトホームアプリとして設定可能

## 技術
- Kotlin / Jetpack Compose / Material 3
- minSdk 29 / targetSdk 36 / 単一モジュール構成
- Room・Hilt不使用（SharedPreferencesのみ）

## ビルドと実行の手順（はじめての方向け）

### 1. Android Studio をインストール
[https://developer.android.com/studio](https://developer.android.com/studio) から「Android Studio」をダウンロードしてインストールします。
インストール時のセットアップウィザードで **Android SDK** が自動で入ります（これにビルドに必要なものが含まれます）。

### 2. プロジェクトを開く
Android Studio を起動 →「Open」→ `C:\AndroidProjects\MinimalLauncher` を選択して開きます。
初回は「Gradle Sync」が走り、必要な部品が自動ダウンロードされます（数分かかります。ネット接続が必要）。

> もし「Android SDK location」を聞かれたら、画面の指示に従ってSDKをインストールしてください。
> もし「Android Gradle Plugin のアップグレードを推奨」と出たら、提案に従って更新してOKです。

### 3. 実機またはエミュレータを用意
- **エミュレータ**: 上部メニュー「Device Manager」から仮想デバイスを作成（Pixel系 + API 34〜36 推奨）
- **実機**: USBデバッグを有効にしたAndroid端末をUSB接続

### 4. 実行
上部の緑の ▶（Run）ボタンを押すと、ビルドして端末にインストール・起動されます。

### コマンドラインでビルドする場合（任意）
Android Studio無しでビルドしたい場合は、JDK 17 と Android SDK を用意し、
プロジェクト直下で次を実行します（Windows）:

```
.\gradlew.bat assembleDebug
```

生成物: `app\build\outputs\apk\debug\app-debug.apk`

## デフォルトホームアプリに設定する方法
アプリを一度起動した後、ホームボタンを押すと「ホームアプリの選択」が出ます。
出ない場合は端末の「設定 → アプリ → デフォルトのアプリ → ホームアプリ」から
「ミニマルランチャー」を選択します。

### 元のランチャーに戻したいとき
同じ「設定 → デフォルトのアプリ → ホームアプリ」から、元のランチャー（Pixel Launcher 等）を選び直してください。

## 使い方
- **アプリを追加/削除**: ホーム画面を長押し、または右上の歯車ボタンから設定画面へ。チェックで追加、チェックを外すと確認後に削除。
- **アプリを起動**: ホーム画面でアイコンをタップ。
- **並べ替え**: ホーム・フォルダ・下段ドックのアイコンを長押ししてドラッグ。
- **その他の設定**: アプリ選択画面の右上にある歯車から、ホームの文字色、JSONバックアップ、起動理由の履歴を操作。
